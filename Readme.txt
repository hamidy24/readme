# Git & GitHub Practice Exercise

## Objective
The primary goal of this exercise was to gain practical experience with version control using Git and GitHub. This involved creating a new GitHub repository, adding an initial C++ file, and managing code changes through a typical Git workflow (clone, modify, stage, commit, and push).

## Learning Outcomes
Through this practice, the following fundamental Git workflows and commands were mastered:

Repository Management: Creating a remote repository on GitHub and linking it to a local environment.
git clone: Downloading the remote repository to the local machine.
git status: Monitoring the current state of the working tree and staging area.
git diff: Inspecting the specific line-by-line modifications made to the code before staging them.
git add: Staging modified files (such as `main.cpp`) to prepare them for a commit.
git commit: Saving a snapshot of the staged changes with a descriptive and meaningful message.
git push: Uploading the local commits to synchronize with the remote GitHub repository.

## Final Result
By the end of the exercise, a new repository named git-practice was successfully established. The repository contains an updated version of `main.cpp`, modified locally to include personal student details (Name, Student ID, Section), and seamlessly pushed back to the remote server.
